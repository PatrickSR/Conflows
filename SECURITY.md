# 安全规范

## 开源前检查清单

在将代码推送到公开仓库前，请确保：

### ✅ 代码检查

- [ ] 移除所有硬编码的 API 密钥、Token、密码
- [ ] 移除公司内部域名（如 `*.company.com`）
- [ ] 移除内部系统名称和 IP 地址
- [ ] 移除真实的业务数据示例
- [ ] 检查所有注释和文档中的示例

### ✅ 文件检查

- [ ] `.env` 文件已在 `.gitignore` 中
- [ ] 测试数据目录已在 `.gitignore` 中
- [ ] 没有提交包含敏感信息的测试文件

### ✅ 依赖检查

- [ ] 所有依赖都是公开可用的
- [ ] 没有使用内部 npm registry

## 搜索敏感信息

在提交前运行以下命令检查：

```bash
# 检查可能的敏感域名
grep -r "\.com\|\.cn\|\.io" src/ --include="*.ts" --include="*.md"

# 检查可能的内部关键词
grep -r "internal\|private\|secret\|key\|token\|password" src/ --include="*.ts" -i

# 检查中文注释中的公司信息
grep -r "公司\|内部\|保密" src/ --include="*.ts"
```

## 数据处理

本项目处理的数据：

- ✅ **本地文件**：工作流 markdown 文件（存储在用户本地）
- ✅ **无网络请求**：不发送任何数据到外部服务器
- ✅ **无遥测**：不收集用户使用数据

## 报告安全问题

如果发现安全问题，请通过 GitHub Issues 报告。
